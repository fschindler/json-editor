# Date Formats

> _json-schema_ offers a set of _date_-format validations per default that are listed below. While you can always add your own (date) validation methods, the widgets of [@sagold/rje-mantine-widgets](?path=/docs/packages-rje-mantine-widgets-overview--docs) will only support the default formats for now.

Thus, working with dates and default date-based features your data needs to rely on one of the three stated formats:

|   type    | format                   | example                  |
| :-------: | :----------------------- | :----------------------- |
|   date    | YYYY-MM-DD               | 2023-08-20               |
|   time    | HH:mm:ssZ                | 16:55:02Z                |
| date-time | YYYY-MM-DDTHH:mm:ss.sssZ | 2023-08-20T14:56:17.657Z |

The _date-time_-format is a [ISO 8601 date-time string format](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Date/toISOString) in UTC timezone and can be retrieved in javascript by `(new Date()).toISOString()`.

An empty string will be accepted. To set a date-format as mandatory add an additional validation `minLength: 1`

```json
{
  "type": "string",
  "format": "date-time",
  "minLength": 1
}
```
